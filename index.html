Important Concepts

Lesson 1: Getting Started with HTML

WEB & HTML Basics

The baiscs of the WEB are broken into 5 categories; WEB, HTML, URLS, HTTP, and WEB Applications. WEB interaction involves computers, HTTP requests, servers, HTML documents, and web browsers. The WWW is a collection of HTML documents, HTML stands for Hypertext Markup Language, it glues everything together and is the heart of the WEB. Hyperlinks link the HTML documents together, making the WEB, web like. The Internet was invented in 1990, it has over 30 billion pages. This video gives a good overview.

HTML, TAGS, & ELEMENTS

HTML documents contain text content which describes "what you see", markup which describes "how it looks", reference to other documents images and videos, and links to other pages. HTML uses tags to tell browsers what type of HTML element to use. This video gives a good overview. HTTP Elements include the opening tag, content, and closing tag. Tags that have content require a closing tag, if the tag does not have content it is known as void and does not require a closing tag.

White Space, Inline, Block

The size of white space always equals one, no matter how large or long the white space appears in HTML. Two Tags that help break out or group the HTML content are line break and paragrah. In addiiton HTML Elements are either inline or block. Block elements form an "invisible box" around the content inside of them. Break is inline and paragrah is block. This video explains the distinction well.

Lesson 2: Creating a Structured Document with HTML

HTML & CSS Defined

A good way to describe HTML and CSS is to use them in an analogy describing a house. HTML is the structure and CSS is the style of the house. HTML and CSS are both languages, HTML controls the structure & content of a web page, CSS controls the style of a web page or "how it looks".

DOM "Tree Like Structure"

DOM stands for Document Object Model, it is a tree like structure were elements branch to other elements. It is the way the computer ties several elements together, resembles a family tree. When Programers talk about the DOM, they are talking about the tree like structure of a page. In the browser, this structure shows up as a series of boxes.

Indentations and HTML

There are several Text editors available for writing code in HTML, we will be using Scratchpad, Codepen, and Sublime for this course. Scratchpad is designed for beginners with less features, Codepen is for users with intermediate experience with additional features, and Sublime is for users with advanced experience with the best features of all three.

Indentations

When you look at a HTML document you see several indentations on the left side of the page. The more severe the indention, the more deeply the element is.

Boxes Everywhere

Every object on a website is a box, even if the object looks like a circle. The HTML structure defines the object as a box, the CSS style defines what is see within the box. There are rules that define how the boxes work within a grid system. Using place holder contacts with in div tags help keep the boxes in order, using logical class names helps add CSS.

Lesson 3: Adding CSS Style and HTML Structure

CSS

CSS means cascading style sheets, cascading is the most important part of the acronym. Known as a search and replace tool, it uses cascading rules on direct mathes and child elements. The idea of cascading is know as inheritance and you need abstract thinking to understand it. CSS is code written to control the style of HTML elements.

Avoiding Repetition

Avoiding repetition is a critical step in creating a solid and efficent CSS code. It helps avoid errors that occur when you perform the same task repetitively and it also helps build consistency which allows efficent changes.

Verification

Using validaiton tools is a critical step of ensuring you have produced valid HTML and CSS coding. The W3C tool Markup Validation Service checks your HTML and the W3C tool CSS Validation Service checks your CSS.
